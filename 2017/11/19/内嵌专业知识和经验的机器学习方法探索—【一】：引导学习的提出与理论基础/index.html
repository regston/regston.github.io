内嵌专业知识和经验的机器学习方法探索—【一】：引导学习的提出与理论基础

发表于 2017-11-19 | 分类于 电力系统 ， 机器学习
字数统计: 1,132 | 阅读时长 ≈ 4

摘要

​ 简要介绍机器学习的发展历程，分析了典型机器学习算法的主要特征及局限性。然后，结合电力领域特征，分别从工程实际、研究机理和“知识资产管理”需要 3 个层面，探讨了电力领域对机器学习方法的要求。在此基础上通过将专业知识和经验进行数学化表示和封装，提出一种嵌入“知识函数单元”的机器学习方法——引导学习，重点研究其理论基础、架构和原理。引导学习的主要特点是结合了领域知识经验和机器学习，提供了一种知识分析与数据挖掘相融合的机器学习范式，探索了人机协同混合增强智能的实现机理和电力知识资产传承管理的可行路线。最后，探讨了需进一步研究的问题。

关键词：引导学习；知识；经验；机器学习；人工智能；智能电网；健康诊断

​ Alpha Go 击败围棋高手、无人驾驶汽车上路、医疗机器人 Watson 诊断出世界稀有病例，标志着人工智能，特别是机器学习(machine learning，ML)技术正在走向主流，备受各行业关注。与此同时，电力系统承载着越来越多的可再生新能源和电动汽车等主动式负荷，正在发展为信息与物理系统高度融合的能源互联网，不断产生维度高、类型广、体量大的数据资源。传统地单纯依靠物理建模分析的理念方法，已无法充分适应智能电网高维、时变、非线性、受内外部多物理参量(环境、气候、工况等)影响所带来的综合挑战$^{[3-5]}$。

​ 如何在电力系统管控各环节，全面有效利用 ML 技术，让计算机从电源出力数据、电网潮流数据、用电负荷数据等诸多实际场景中发现问题、挖掘规律、辅助决策，以支持电力系统规划、调度、检修、营销各生产实践环节，是当前备受电力领域关注的热点难点。

​ 传统 ML 算法主要对数据的搜索、分类、比较、聚类、关联等进行分析和归纳。在电力领域，一些学者应用支持向量机、Logistics 回归等，在新能源发电预测、用户用电行为模式识别等多场景中取得有益结果。然而，当前一些机器学习算法，如深度学习局限性

  1. 其输入一般需要足量且有效的训练样本。而在诸如电网故障诊断等应用场景中，获得足量的表征电网异常状态的样本较困难，且通常代价昂贵

  2. 其任务多针对边界确定的封闭环境，一旦环境开放、多变，可能导致结果失准、鲁棒性差。而电力系统是开放动态系统，源–荷端皆存在不确定性和随机性，加之运行工况、环境时刻变化，导致现有 ML 算法尚未在诸如电网调度等对安全性、鲁棒性要求高的场景中充分应用

  3. 其算法多为“黑箱模型”，在样本训练过程中，未有效利用电力系统相关知识对学习器进行指导，而仅仅挖掘样本数据的“表象”规律，导致训练的模型往往“只知结果，不明就里”，对电力系统生产实践的指导能力有限。

    ​

​ 一方面，电力领域的知识、经验或经过严格数学论证，或在长期生产实践中形成，实际指导价值巨大；另一方面，ML 能够对数据进行智能分析，善于发现潜在问题、挖掘隐性规律。如果能在 ML 算法中融入一定的知识经验，以引导学习器对数据样本进行挖掘，例如，当学习器沿着正确的方向挖掘规律时予以肯定、强化，反之则进行纠偏、弱化，就可能实现学习器对输入样本要求的降低，对输出结果安全性、鲁棒性等性能的提升，并有望开发出更为通用的 ML 方法、覆盖更广的学习任务、拓宽其在电力领域的应用价值。
